 1 # Git Backup Skill
 2 
 3 ## Description
 4 Automated git backup skill that commits and pushes local paths to a remote git repository. Prioritizes local changes over remote changes while preserving remote content in git history for easy recovery.
 5 
 6 ## Features
 7 - **Configurable backup paths**: Define multiple directories/files to backup
 8 - **Conflict resolution**: Prioritizes local changes using `-X ours` merge strategy
 9 - **History preservation**: Remote content is preserved in git history for recovery
10 - **Automatic staging**: Handles new files, modifications, and deletions
11 - **Error handling**: Comprehensive error messages and status reporting
12 
13 ## Configuration
14 
15 ### Backup Paths
16 Edit the `BACKUP_PATHS` list at the top of the script:
17 
18 ```python
19 BACKUP_PATHS = [
20     "/a0/usr",
21     # Add more paths as needed
22 ]
23 ```
24 
25 ### Remote Repository
26 The script expects each backup path to be a git repository with a configured remote.
27 
28 ```bash
29 cd /your/backup/path
30 git remote add origin <your-repo-url>
31 ```

63 ## Git Strategy
64 
65 The script uses a strategy that **prioritizes local changes** while **preserving remote history**:
66 
67 1. **Stage all changes**: `git add -A` (includes new, modified, and deleted files)
68 2. **Commit locally**: Creates a timestamped commit with all local changes
69 3. **Fetch remote**: Gets the latest remote state without merging
70 4. **Merge with local priority**: `git merge origin/<branch> -X ours`
71    - The `-X ours` flag resolves conflicts by preferring local changes
72    - Remote content is preserved in the merge commit history
73 5. **Push to remote**: Uploads the merged result
74 
75 ### Recovery
76 If the script overwrites something important from remote:
77 1. Check `git log` to find the commit before the merge
78 2. Use `git show <commit>:<file>` to view the old content
79 3. Restore with `git checkout <commit> -- <file>` or cherry-pick
