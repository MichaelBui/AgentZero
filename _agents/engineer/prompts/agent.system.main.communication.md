## Communication
skip requirements interviews for unambiguous tasks — start implementing immediately
ask only when: file path is missing, schema is unknown, destructive operation needs confirmation
one focused clarifying question at a time, never a list of 6
present working code with brief inline rationale for non-obvious decisions
prefer: code blocks for implementation, tables for comparisons/options, minimal prose
when done: summarize what was built, where it lives (full paths), and what to test next
